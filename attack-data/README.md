Raw log files for the "attack data" malware micro-CTF
(`challenges/attack-data-ctf/`), one small scenario per malware family,
downloaded from [splunk/attack_data](https://github.com/splunk/attack_data)
(a Git LFS repo — files fetched via the `media.githubusercontent.com` CDN,
see `docker/download_attack_data.py`, no git-lfs client required).

- `manifest.json` — which file was picked per family, its `sourcetype`/
  `source` (as declared by attack_data's own per-scenario `.yml`), and a
  short description. Generated once by hand; not auto-regenerated by
  `setup.sh` (only the actual `.log`/`.yml` downloads are).
- `<family>/*.yml` — attack_data's own metadata for that scenario
  (kept small, tracked in git).
- `<family>/*.log` — the actual raw event log (10 KB – 75 MB,
  gitignored, downloaded on demand by `./setup.sh --attackdata`).

Picked the **smallest** available scenario per family (attack_data ships
several per malware, often 1–7) to keep total download size sane for a
"micro" pack — 34 families, ~330 MB total. Two families
(`minergate`, `xmrig_miner`) were excluded outright: their only available
scenario is 240 MB / 365 MB respectively.

Ingested into the `attack_data` Splunk index by `setup.sh` via
`splunk add oneshot` (or the equivalent REST call) — see
`docker/apps/attack_data_ingest/` for the index definition and XML field
extractions, and the root README's "Attack data micro-CTF" section for
usage.
